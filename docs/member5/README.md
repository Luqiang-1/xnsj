# 成员5工作文件夹说明

本文件夹用于集中展示成员5在“数据采集、清洗与知识库入库工具”方面完成的工作。正式问答数据仍然来自项目根目录下的 `knowledge/`，本文件夹主要用于给老师查看成员5的工作痕迹、流程说明和文件用途。

## 文件说明

### data_pipeline.md

数据采集、清洗与入库流程说明。

主要介绍成员5如何完成资料检索、来源登记、采集演示、网页噪声清理、Markdown 初稿生成、人工校对衔接和正式入库。适合用来说明成员5的整体工作流程。

### work_trace.md

成员5工作痕迹说明。

主要列出成员5提交了哪些材料，包括采集工具、来源登记表、离线演示网页、清洗规则、采集初稿、运行日志，以及 `knowledge/` 中补充的 Kevin 整理记录。适合老师快速检查成员5的工作量。

### file_guide.md

成员5相关文件和文件夹的详细用途说明。

该文件逐个解释了以下内容：

* `knowledge/`：正式知识库文件夹，已补充整理人员 Kevin 和整理时间。
* `tools/member5_crawler.py`：成员5采集与清洗演示工具。
* `data/member5/source_register.csv`：来源登记表。
* `data/member5/sample_pages/`：离线演示网页。
* `data/member5/cleaning_rules.md`：数据清洗规则。
* `data/member5/drafts/`：采集初稿输出目录。
* `data/member5/logs/crawl_log.jsonl`：采集运行日志。
* `docs/member5/`：成员5文档集中存放目录。

老师如果想详细了解每个文件的作用，可以优先查看 `file_guide.md`。

## 成员5展示时可以这样说

成员5主要负责知识库资料进入系统前的准备工作：先登记资料来源，再使用采集工具提取网页正文，清洗掉导航、版权、重复段落等噪声，然后生成统一格式的 Markdown 初稿。初稿不会直接进入问答系统，需要人工校对后再维护到 `knowledge/`。为了方便追溯，正式知识库文件末尾统一补充了整理人员 Kevin 和整理时间。
