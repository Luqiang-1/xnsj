# 成员5：数据采集、清洗与知识库入库流程

## 数据来源方案

正式问答数据直接使用 `knowledge/` 目录中的 Markdown 文件。为了展示成员5的采集与清洗工作，项目保留了 `tools/member5_crawler.py` 和 `data/member5/` 下的离线演示材料。

成员5保留“采集痕迹”和“清洗痕迹”：说明资料可来自农业农村部门、农技推广中心、高校或科研机构、标准信息平台、公开扩展服务网站等公开页面；如果后续使用真实网页爬虫，只采集公开、允许访问的页面，并记录来源链接，不采集需要登录、付费、绕过权限或 robots.txt 明确禁止的内容。

## 采集与清洗流程

1. 资料检索：围绕 `knowledge/` 中已有作物，查找公开农业技术资料、地方标准、病虫害防控资料和采收销售建议。
2. 采集留痕：在 `data/member5/source_register.csv` 记录来源网站、来源链接、采集时间和整理人员。
3. 工具演示：运行 `python tools/member5_crawler.py --sources data/member5/source_register.csv`，生成采集日志和 Markdown 初稿。
4. 清洗留痕：说明已删除网页导航、广告、版权栏、上一篇/下一篇、重复段落、联系方式和与作物无关的介绍。
5. 结构整理：把有效内容放入“品种介绍、种植方式、环境需求、常见病虫害与解决方案、农药使用、采收与销售、常见问题、资料来源”等统一小节。
6. 人工校对：重点核对温度、湿度、土壤、病虫害、农药登记范围、安全间隔期、采收贮运等信息。

## Markdown 入库流程

`data/member5/drafts/` 只作为成员5采集初稿展示目录，不直接参与 RAG 问答。正式入库时，把成员2校对后的内容维护到 `knowledge/分类/作物.md` 中。

修改 `knowledge/` 后，服务运行时访问 `http://127.0.0.1:8000/api/reload` 重新加载知识库，再使用正例和反例问题检查检索效果。

## 来源记录格式

每个作物文件末尾保留 `## 资料来源`，建议包含：

* 来源网站
* 来源链接
* 采集时间
* 整理人员：Kevin
* 整理时间：2026-05-19
* 校对状态
* 生产提示：具体农药和剂量以当地农技部门指导、农药标签和现行标准为准

## 后续扩展设计

后续可以在后台增加知识库管理功能：查看已加载文件、展示每个文件的来源和整理记录、点击重新加载知识库、对重复标题或空章节进行提示。这样成员5负责采集和清洗留痕，成员2负责内容质量，成员4负责检索效果，流程会更清晰。
